# 未明文芸会資料室 プロトタイプ

ローカル確認用の資料室サイトです。現在は依存関係なしで起動できる `preview-server.mjs` を主実装として使っています。

## 起動

```powershell
& 'C:\Users\soshi\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe' preview-server.mjs
```

既定のURLは `http://localhost:3000/` です。

## 公開とGoogle掲載

Google検索に載せるには、固定のHTTPS URLで公開したうえで、Search Consoleにサイトを登録し、`/sitemap.xml` を送信します。

公開候補:

- Render / Railway / Fly.io などのNodeサーバー対応ホスティング: `npm start` で `preview-server.mjs` を起動。
- VPS: Node.jsを入れて `npm start` を常駐。

公開後に確認するURL:

- `/`
- `/robots.txt`
- `/sitemap.xml`

Search Consoleに送信するsitemap:

```text
https://公開ドメイン/sitemap.xml
```

検索に出したくない内部ページは、HTML側で `noindex,nofollow` を付与しています。

## 正解ルート

1. 外部入口から資料室へ来る。
2. 既刊一覧で第17号が表示されていないことに気付く。
3. 更新履歴で `MR-2020-0612` を見つける。
4. 通常検索にMR番号を入力し、管理用ページで確認する案内を見る。
5. フッターの小さな「管理用」から `/admin` へ進む。
6. `MR-2016-0509` または `MR-2023-0320` を照会し、関連文書として部内通知 `2012-03` を見つける。
7. 部内通知 `2012-03` から、非公開資料に移行した部誌本体が `A-410` 系で管理されることを知る。
8. 第17号は再登録されていないため、`A-410-17` と推測する。
9. 管理用ページで `A-410-17` を照会し、制限資料カードを見る。
10. 照会IDが必要だと分かる。
11. 部内通知番号の並びから `2012-04` を推測し、管理用ページで直接照会する。
12. PBが発行責任者、関連号数、発行時点の担当者番号で照会IDを作ることを知る。
13. 発行時点編集体制を検索し、2019年度後期〜2020年度前期の発行責任者が佐伯亮、担当者番号01であることを確認する。
14. `A-410-17` の制限資料カード内で `PB-17-01` を入力する。
15. 第17号が表示され、三橋澪「午前二時のコピー機」と奥付の発行責任者 佐伯亮に気付く。

## 各ページの役割

- `/`: 資料室の入口。公開ナビのみ表示。
- `/about`: 資料室の概要。
- `/issues`: 公開版として登録された既刊一覧。第17号は表示しない。
- `/issues/17`: 公開停止ページ。資料番号や照会IDは出さない。
- `/updates`: 通常登録履歴とMR処理記録番号の発見場所。
- `/members`: 公開部員記録。
- `/members/*`: 個別人物記録。第17号や核心作品名は出さない。
- `/search`: 公開資料向け検索。MR、部内通知、A-410系は本文を出さず管理用へ案内する。
- `/admin`: 管理用照会。MR、部内通知、A-410-17制限資料カードを扱う。
- `/material-numbers`: 資料番号の一般説明。A-410や第17号周辺の具体例は出さない。
- `/editorial-structure`: 発行時点編集体制。検索経由のみ。
- `/graduates`: 卒業生記録。検索経由のみ。

## 実装できたこと

- トップナビを公開ページ5件だけに整理。
- 全ページ共通フッターに小さな「管理用」リンクを追加。
- `/admin` に資料管理照会フォームを実装。
- 通常検索ではMR番号、部内通知番号、A-410系、照会ID本文を直接表示しないように変更。
- 管理用ページでMR-2012-0405、MR-2016-0509、MR-2020-0612、MR-2023-0320、MR-2023-0430を照会可能にした。
- 部内通知2012-01〜2012-05を管理用ページで個別照会可能にした。
- A-410-17カード内の照会ID欄でPB-17-01を入力した場合のみ第17号を表示。
- 個別人物ページの「在籍記録」を「公開部員記録」に置換。
- 人名検索・作品名検索で第17号情報が漏れないように調整。
- 404を資料室内検索の失敗画面風に整理。
- sitemapを入口ページだけに限定し、内部資料や検索結果にはnoindexを付与。

## 仕様から判断して補ったこと

- `/admin` と `/manage` は同じ管理用ページを表示するようにした。
- 旧 `/notices/*`、`/access-id`、`/lookup`、`/restricted` は資料室風404へ回した。
- 通常検索で部内通知の内容語が入力された場合は、本文ではなく管理用ページへの参考候補だけを返すようにした。
- `A-410-09` や `A-410-23` などA-410系の具体番号も通常検索では管理用案内に寄せた。
